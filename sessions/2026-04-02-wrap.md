## Decisions Made This Session

### LLM
- **Claude 3 Haiku** via Amazon Bedrock. Chosen for speed (< 2 seconds typical), low cost ($0.25/1M input tokens), and high quality for factual Q&A on retrieved context.

### Embeddings
- **Amazon Titan Embeddings v2** (`amazon.titan-embed-text-v2:0`). AWS-native, no external API key, 1536-dimension output, consistent with Bedrock authentication.

### Vector Store
- **S3 (JSON files)**. No additional service (e.g., OpenSearch, Pinecone) needed at demo scale. Vectors stored as JSON objects alongside chunk text.

### Retrieval Method
- **Custom cosine similarity in Lambda**. Explicitly NOT using Bedrock Knowledge Bases. The custom layer demonstrates engineering skill and keeps the retrieval logic visible, testable, and explainable.

### Infrastructure
- **CloudFormation** for all MVP resources. Ensures infrastructure is version-controlled, reproducible, and reflects real-world AWS engineering practice.

### Observability
- **CloudWatch** (logs + dashboard) + **CloudTrail** (audit trail) + **AWS Budgets** (cost alerts). All AWS-native. CloudTrail first trail is free. Budgets first two alerts are free.

### Phase 2 deferral
- **DynamoDB, Cognito, CloudFront** deferred to Phase 8. Not needed for a functional portfolio demo. Adding them in MVP would add complexity without proportional value.

### Frontend
- **Static HTML/CSS/JS on S3**. No framework needed. S3 static website hosting is free at demo scale.

### Cost cap
- **$20/month hard cap**. Estimated actual spend: $2–5/month at demo traffic. Bedrock is the only meaningful cost driver.

### AWS Region
- **us-east-1**. Best Bedrock model availability.
