What is RAG?

RAG stands for Retrieval-Augmented Generation. Source documents are split into small chunks and each chunk is converted into a vector — a list of numbers that captures its meaning — using Amazon Titan Embeddings. When you ask a question, the question is embedded the same way, and cosine similarity finds the chunks whose vectors are closest to the question. Those matching chunks are passed to a large language model as context, so the answer is grounded in real source material rather than the model's general training data.

How It's Built

  • Embeddings: Amazon Titan Embeddings v2 — 1536-dimension vectors stored as a JSON index in S3
  • Retrieval: Cosine similarity search running inside AWS Lambda over the S3 vector index — no managed vector DB
  • Generation: AWS Bedrock — Claude Haiku 4.5 via cross-region inference profile
  • Infrastructure: API Gateway, Lambda, S3, CloudFront, CloudFormation IaC
  • Observability: CloudWatch dashboard, p95 latency + error-rate alarms, SNS alerts
